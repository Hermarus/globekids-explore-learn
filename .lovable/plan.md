

# Отправка заявок в Telegram через бота

## Что будет сделано
При отправке формы на сайте заявка автоматически отправляется сотруднику в Telegram через бота. Секреты (токен бота и chat ID) уже сохранены -- осталось создать серверную функцию и подключить форму.

## Текущее состояние
- Lovable Cloud подключён (это ваш бэкенд, аналог Supabase)
- Секреты `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` уже сохранены
- Edge function ещё не создана
- Форма пока не отправляет данные

## Шаги реализации

### 1. Создать серверную функцию `send-telegram`
Файл: `supabase/functions/send-telegram/index.ts`

Функция будет:
- Принимать POST-запрос с данными формы (имя, телефон, email, ребёнок, возраст, направление)
- Читать секреты `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` из окружения
- Форматировать красивое сообщение
- Отправлять его через Telegram Bot API (`sendMessage`)
- Возвращать результат на фронтенд
- Включать CORS-заголовки для работы с веб-приложением

Также добавить в `supabase/config.toml` настройку `verify_jwt = false` для этой функции (заявки публичные, авторизация не нужна).

### 2. Обновить форму заявки
Файл: `src/components/ApplicationForm.tsx`

Изменения:
- Добавить состояние загрузки (`isLoading`)
- При отправке формы вызывать серверную функцию через `supabase.functions.invoke('send-telegram', { body: formData })`
- Показывать спиннер/текст "Отправка..." на кнопке во время загрузки
- Показывать успех только при реальной отправке
- При ошибке -- показывать уведомление (toast) и не скрывать форму

### Формат сообщения в Telegram

```text
Новая заявка с сайта!

Родитель: Иван Иванов
Телефон: +7 (800) 123-45-67
Email: ivan@mail.ru
Ребёнок: Маша, 12 лет
Направление: Дубай, ОАЭ
```

## Техническая архитектура

- Фронтенд (ApplicationForm.tsx) вызывает серверную функцию через SDK
- Серверная функция (send-telegram) отправляет POST в Telegram Bot API
- Токен бота хранится в секретах, не в коде
